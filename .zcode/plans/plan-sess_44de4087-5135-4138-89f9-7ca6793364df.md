# Car Dealership — Full Feature Implementation Plan

A complete, integrated full-stack implementation of all 10 features using **React + JavaScript (frontend)** and **Node.js/Express + MongoDB (backend)** — no TypeScript. Everything is wired end-to-end so the website works as a whole.

I will follow existing conventions exactly: ESM modules with `.js` extensions, named exports for controllers/services, default exports for models and React pages, Tailwind utility classes, axios `api` instance with JWT interceptor, `alert()`-style error handling replaced by toasts, and `{ message }` JSON error shapes.

## Phase 1 — Cross-cutting Foundation (enablers for a "complete" site)

### A. React Auth Context (`frontend/src/context/AuthContext.jsx`)
The app currently reads `localStorage.getItem("token")` once at render, forcing hard reloads on login/logout, and the Navbar shows Admin/Profile to everyone. I'll add a lightweight context providing `{ user, isAdmin, login(), logout(), updateUser() }` backed by localStorage. This enables:
- Reactive Navbar (show **Admin** + **My Orders** only to admins/logged-in users, hide on login page).
- Client-side route guards (so non-admins can't reach `/admin/*` — backend already enforces).
- `Navbar` logout properly clears both `token` and `user`.

**Files touched:** new `context/AuthContext.jsx`; wrap `<App>` in `main.jsx`; update `App.jsx` (use `useAuth` instead of raw localStorage), `Navbar.jsx`, `Login.jsx`, `Register.jsx`, `Profile.jsx`.

### B. Toast Notifications (Feature 8) + Loading Spinner (Feature 7)
Do these early so every other feature uses them.

- **Feature 8:** install `react-toastify`. Add `<ToastContainer />` in `main.jsx`. Replace **every** `alert(...)` with `toast.success(...)`/`toast.error(...)` across Login, Register, AddVehicle, EditVehicle, VehicleDetails, Profile, AdminVehicleTable (+ new pages).
- **Feature 7:** new `components/Spinner.jsx` (Tailwind `animate-spin` ring). Replace all `Loading...`/`Loading Vehicles...` text in VehicleDetails, Vehicles, MyOrders, AdminOrders, AdminDashboard.

## Phase 2 — Backend: New Models, Controllers, Services, Routes

### New files
| File | Purpose |
|---|---|
| `backend/models/Purchase.js` | `{ userId, vehicleId, vehicleSnapshot:{name,brand,image}, price, quantity, date }` (Feature 2/3) |
| `backend/controllers/purchaseController.js` | `getMyOrders`, `getAllOrders` (admin) |
| `backend/services/purchaseService.js` | DB queries + create-on-purchase logic |
| `backend/routes/purchaseRoutes.js` | `GET /api/purchases/my-orders` (protected), `GET /api/purchases` (admin) |
| `backend/controllers/statsController.js` | `getStats` → counts |
| `backend/services/statsService.js` | `Vehicle.countDocuments`, `User.countDocuments`, `Purchase.countDocuments`, `Vehicle.countDocuments({quantity:0})` |
| `backend/routes/adminRoutes.js` | `GET /api/admin/stats` (protected+admin) |
| `backend/middleware/uploadMiddleware.js` | multer (memory storage), `upload.single("image")` |
| `backend/config/cloudinary.js` | conditional Cloudinary config (only if env vars present) |
| `backend/utils/handleImageUpload.js` | if Cloudinary configured → upload buffer → return `secure_url`; else write to `backend/uploads/<name>` → return `/uploads/<name>` |
| `backend/scripts/seedAdmin.js` | creates default admin `admin@cardealer.com` / `admin123` (idempotent) |

### Modified backend files
- **`vehicleService.js`** → `purchaseVehicle`: also create a `Purchase` record (snapshot name/brand/image/price, userId from caller, quantity 1). Add pagination+search params support to `getVehicles`/`searchVehicles`.
- **`vehicleController.js`** → `createVehicle` & `updateVehicleDetails`: accept `multipart/form-data`, run multer, call `handleImageUpload`, merge returned URL into body before save. Add `page`/`limit` passthrough.
- **`vehicleRoutes.js`** → attach `upload.single("image")` to POST/PUT; reorder search route; add pagination.
- **`userController.js`** + **`userRoutes.js`** + **`userService.js`(frontend)** → add **Change Password**: `PUT /api/users/password` (verify current via `bcrypt.compare`, hash new, save).
- **`app.js`** → register `purchaseRoutes`, `adminRoutes`; add `app.use("/uploads", express.static("uploads"))` for local image fallback; keep `cors()`.

### API shape decisions (backward compatible)
- `GET /api/vehicles` → keep returning **array** when no pagination params (so Home Featured/Latest + Admin table keep working). When `?page=&limit=` present → return `{ vehicles, currentPage, totalPages, totalVehicles }`.
- `GET /api/vehicles/search` → supports `brand,name,category,fuel,transmission,year,minPrice,maxPrice,sort,page,limit` and returns the paginated object. Used by the Vehicles page (Feature 10).
- `GET /api/admin/stats` → `{ totalVehicles, totalUsers, totalOrders, outOfStock }`.
- `GET /api/purchases/my-orders` → array of user's purchases; `GET /api/purchases` (admin) → all purchases with user name populated.

## Phase 3 — Frontend: Pages & Components per Feature

### Feature 1 — User Profile (enhance) + Change Password

- New route `/orders`. Navbar gets **My Orders** link (visible to logged-in users).
- `VehicleDetails.jsx` purchase flow → toast success, then navigate to `/orders`.

### Feature 3 — Admin Purchase Management
- New **`pages/AdminOrders.jsx`**: admin table — **who purchased** (user name), vehicle, date, quantity, price.
- New route `/admin/orders`.
- **`AdminDashboard.jsx`**: add tab/link navigation between **Manage Vehicles** and **Orders**. Admin link in Navbar only for `role==='admin'`.

### Feature 4 — Dashboard Statistics (real numbers)
- New **`services/statsService.js`**: `getStats()`.
- **`AdminDashboard.jsx`**: top row of 4 stat cards (Total Vehicles, Total Users, Total Orders, Out of Stock) fetched from `GET /api/admin/stats` — replaces the bare header. (Home-page `Statistics.jsx` marketing block left as-is; the wireframe targets the dashboard.)

### Feature 5 — Image Upload (Cloudinary + local fallback)
- `AddVehicle.jsx` & `EditVehicle.jsx`: replace image text input with `<input type="file" accept="image/*">`, show preview, send `FormData` (image file + text fields).
- `vehicleService.js` → `createVehicle`/`updateVehicle`: send FormData, set `Content-Type: multipart/form-data`.
- New **`utils/imageUrl.js`**: `getImageSrc(image)` — returns full URL for Cloudinary (`http...`), `/uploads/...` for local, or `/images/<name>` for legacy bare filenames.
- Apply `getImageSrc()` in `VehicleCard`, `VehicleDetails`, `AdminVehicleTable`, `MyOrders`, `AdminOrders` so old seeded vehicles (filenames) and new uploads both render.
- `.env` gets optional `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET` (with placeholders + comments). Works immediately via local fallback until you add Cloudinary creds.

### Feature 6 — Pagination
- **`Vehicles.jsx`**: add `page`/`totalPages` state, **Prev/Next** buttons + "Page X of Y", request `?page=&limit=12` (or backend search). Combined with Feature 10.
- `vehicleService.js` → add `getVehiclesPage(page, limit)` returning the paginated object.

### Feature 9 — Confirm Delete Modal (+ Restock modal)
- New **`components/ConfirmModal.jsx`**: reusable yes/no modal (title, message, onConfirm, onCancel) with Tailwind overlay.
- New **`components/RestockModal.jsx`**: modal with number input (replaces `prompt()`).
- **`AdminVehicleTable.jsx`**: use `ConfirmModal` for delete, `RestockModal` for restock. (Removes the last `window.confirm`/`prompt` in the app.)

### Feature 10 — Search from Backend
- `vehicleService.js` → `searchVehicles(params)`: `GET /api/vehicles/search` with query string.
- **`Vehicles.jsx`**: replace in-memory `.filter()` with backend search calls (debounced ~300ms via a small `useDebounce` hook). `VehicleFilter` stays as the UI; filter changes trigger backend queries. Works together with pagination.
